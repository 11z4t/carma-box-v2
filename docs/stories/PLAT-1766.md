# PLAT-1766 — SoC Need-Based Proportional Charge Distribution

**Status:** Implementation
**Branch:** fix/PLAT-1766-need-based
**Priority:** HIGH — active SoC de-sync live (+5 pp)

---

## Problem

PLAT-1755 introduced cap-proportional weighting in `_distribute()` (balanced
path). For a kontor (15 kWh) + förråd (5 kWh) setup the split is:

```
kontor: 15/(15+5) = 75 %   →  915 W
forrad:  5/(15+5) = 25 %   →  471 W
```

Both banks charge at the same SoC rate (pp/h) relative to their capacity.
This is correct for **maintaining** a balanced state but **wrong for converging**
when one bank is already ahead.

### Live example (2026-04-22)
```
kontor SoC: 46 %  (15 kWh)
forrad SoC: 51 %  (5 kWh)   ← 5 pp ahead
surplus: 1032 W

PLAT-1755 allocation:
  kontor: 1032 * 0.75 = 774 W  →  774 / 15 000 Wh = 0.0516 pp/h
  forrad: 1032 * 0.25 = 258 W  →  258 /  5 000 Wh = 0.0516 pp/h

SoC rates are identical → diff stays at 5 pp forever.
```

PLAT-1715 requires SoC diff ≤ 1 pp.

---

## Solution

### Algorithm — need-based weights (balanced path only)

**Charging** (`total_target_net_w < 0`):

```python
weight[b] = max(0.0, (1 - soc[b] / 100) * cap_kwh[b])
```

A battery with 46 % SoC has need 0.54 × 15 = 8.1 kWh.
A battery with 51 % SoC has need 0.49 × 5  = 2.45 kWh.

Split: 8.1/(8.1+2.45) = 76.8 % kontor vs 23.2 % förråd.
At 1032 W: kontor=793 W (0.053 pp/h), forrad=239 W (0.048 pp/h).
Kontor charges faster → converges.

**Discharging** (`total_target_net_w > 0`):

```python
weight[b] = max(0.0, (soc[b] / 100) * cap_kwh[b])
```

Fuller battery discharges more → protects the emptier one.

### Fallback when all weights = 0

- Charging fallback: all bats at 100 % SoC — weight=0 for everyone.
  Falls back to PLAT-1755 cap-proportional weighting.
- Discharge fallback: all bats at 0 % SoC — same fallback.

### Feature flag

`BudgetConfig.bat_need_based_enabled: bool = False` (default OFF).
`BudgetSection.bat_need_based_enabled` in site.yaml.

When `False`: PLAT-1755 cap-proportional is used (unchanged behaviour).
When `True`: need-based weights applied in balanced path.

### Aggressive path NOT affected

The aggressive split (SoC spread > `bat_aggressive_spread_pct`) already
directs charge to the emptier battery exclusively. It is untouched.

---

## Edge Case Matrix

| Scenario | Behaviour |
|----------|-----------|
| Single battery | Unchanged — gets full target |
| All bats at 100 % SoC (charge) | weights=0 → fallback to cap-proportional |
| All bats at 0 % SoC (discharge) | weights=0 → fallback to cap-proportional |
| One bat at SoC max (charge) | weight=0 for that bat → fully excluded |
| Asymmetric caps + asymmetric SoC | Both factors combined in weight |
| Spread > aggressive threshold | Aggressive path unchanged |
| `need_based=False` | Identical to PLAT-1755 |
| Single bat at exact 100 % | weight=0, only other bats get charge |
| cap_kwh=0 (legacy) | weight=0 → that bat excluded from need-based; fallback if all zero |

## Acceptance Criteria

1. `bat_need_based_enabled=False` (default) → identical output to PLAT-1755 (regression-free)
2. `bat_need_based_enabled=True` + asymmetric SoC → lower-SoC bat gets higher charge share
3. All-full fallback → cap-proportional (no divide-by-zero)
4. Discharge: higher-SoC bat gets larger share
5. Aggressive path unaffected by flag
6. 25+ tests pass, mypy --strict 0 errors, ruff 0 errors
7. PLAT-1715 1 pp tolerance: starting at 5 pp diff, need-based converges within 30 min at realistic charging

## Convergence Analysis

At 1032 W surplus, need-based split:
- kontor: 793 W → 0.053 pp/h  (15 kWh cap)
- forrad: 239 W → 0.048 pp/h  ( 5 kWh cap)

Net convergence: 0.053 - 0.048 = 0.005 pp per cycle (30 s).
Per hour: 0.005 × 120 cycles = 0.6 pp/h.

Starting at 5 pp diff → converges in ~8.3 hours at steady 1032 W.

At higher surplus (e.g. 3000 W):
- Need-based split (76.8 % / 23.2 %): kontor=2304 W, forrad=696 W
- Rate kontor: 2304/15000 × 100 = 15.4 pp/h
- Rate forrad:  696/ 5000 × 100 = 13.9 pp/h
- Net convergence: 1.5 pp/h → 5 pp resolved in ~3.3 hours

As the gap narrows, weights equalise → convergence slows naturally (self-limiting).
