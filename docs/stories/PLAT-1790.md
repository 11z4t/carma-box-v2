## 1. Problemanalys & Root Cause

### 1.1 Observerade symptom (2026-04-22→23)

| Datum | Symptom |
|-------|---------|
| 2026-04-20→21 | 1 220 events på `switch.easee_home_12840_is_enabled` |
| 2026-04-22→23 | Ingen EV-laddning trots v2-writes |
| 2026-04-23 | Switch oscillerar 30 ggr/2h när bil EJ är inkopplad |

### 1.2 Root cause — bekräftat

**Primär: Tyst code-path i plan executor (coordinator.py:1473–1481)**

```python
# coordinator.py:1473–1481 (NUVARANDE — BUGGY)
if self._night_ev_active:
    pass  # Night EV has control — don't override
elif cmd.ev_action == "start" and cmd.ev_amps >= DEFAULT_EV_MIN_AMPS:
    if not self._ev_enabled:
        await self._cmd_ev_start(cmd.ev_amps)  # ← SKRIVER UTAN CABLE CHECK
    elif cmd.ev_amps != self._ev_current_amps:
        await self._cmd_ev_adjust(cmd.ev_amps)  # ← SKRIVER UTAN CABLE CHECK
elif cmd.ev_action == "stop" and self._ev_enabled:
    await self._cmd_ev_stop()
```

**Problem:** `cmd.ev_action` sätts baserat på plan/pris-logik utan att kontrollera om bilen faktiskt är ansluten. Om plan-executorn säger "start" → skriver v2 till EV även när `cable_locked = False`.

**Sekundär: Ingen idempotence i ev_night_controller**

NEV-state-machine (`core/night_ev.py` + koordinatorns NEV-integration vid rad 1450–1456) emitterar `start_ev` / `start_discharge` kommandon varje cykel utan att kontrollera om vi redan är i rätt tillstånd. Specificerat: ingen check mot `_ev_enabled` eller `_ev_current_amps` INNAN NEV-dispatch.

**Tertiär: Startup recovery skriver direkt till HA-entitet utan cable check (rad 2050–2062)**

```python
# coordinator.py:2050–2062 (NUVARANDE — BUGGY)
if getattr(self, "_night_ev_active", False):
    await self.hass.services.async_call(
        "switch", "turn_on",
        {"entity_id": "switch.easee_home_12840_is_enabled"},  # ← HÅRDKODAT + INGEN CABLE CHECK
    )
```

**Kvartär: `_self_heal_ev_tamper` triggar write-loop**

När Easee cloud eller HA-automation touchar `is_enabled` externt → `_self_heal_ev_tamper` (rad 6681–6704) flaggar tamper → nästa cykel korrigerar v2 via `_cmd_ev_start/stop` → Easee cloud korrigerar tillbaka → oscillation.

### 1.3 Hotfix PLAT-1786 (kvar, otillräcklig)

`EVControllerConfig.start_headroom_w = 1_000_000` — sätter effektivt headroom till oändlighet, vilket förhindrar EV-start men blockerar ALLA laddfall. Löser oscillation via "stänger av allt" men är inte en korrekt fix. **Måste tas bort i Fas 6.**

## 3. State-machine: ev_dispatch_v2

### 3.1 Tillstånd

```
┌─────────────────────────────────────────────────────────────────┐
│                     ev_dispatch_v2 States                        │
│                                                                   │
│   IDLE ──────────────────────────────────────────────────────┐   │
│    │                                                          │   │
│    │ R2: plug connected                                       │   │
│    ▼                                                          │   │
│  CONNECTED                                                    │   │
│    │                                                          │   │
│    │ R12: bat_soc >= bat_ev_ready_threshold                   │   │
│    │ + R10: predicted_refill >= predicted_deficit             │   │
│    │ + R11: planned_window >= 15min                           │   │
│    │ + R9:  bat_soc at forecast-surplus-end == 100%           │   │
│    ▼                                                          │   │
│  ANALYZING                                                    │   │
│    │                                                          │   │
│    │ all acceptance criteria met                              │   │
│    ▼                                                          │   │
│  CHARGING ◄──── bat smoothing active (R3) ────────────────┐  │   │
│    │                                                       │  │   │
│    │ PV-transient detected                                 │  │   │
│    ├──► (stay in CHARGING, bat covers dip)                 │  │   │
│    │                                                       │  │   │
│    │ acceptance criterion lost                             │  │   │
│    ├──► COMPLETING (grace 1 cykel = 15s)                   │  │   │
│    │                                                       │  │   │
│    │ target_soc reached OR window exhausted                │  │   │
│    ├──► COMPLETING ──► IDLE                                   │   │
│    │                                                          │   │
│    │ R2 violated (plug disconnected)                          │   │
│    ├──► ERROR ──► IDLE (after emit stop_write)                │   │
│    │                                                          │   │
│    │ R1 violated (grid > ±100W)                               │   │
│    └──► ERROR ──► IDLE (emit stop_write + incident_alert)  ───┘   │
│                                                                   │
│  ERROR                                                            │
│    │ always → IDLE (after stop_write emitted)                     │
│    └──────────────────────────────────────────────────────────────┘
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 Transitions (detaljerad)

| Från | Till | Trigger | Guard | Action |
|------|------|---------|-------|--------|
| IDLE | CONNECTED | `ev_status ∈ {connected,awaiting_start,charging,ready_to_charge}` | — | emit `log_connected` |
| IDLE | IDLE | `ev_status == disconnected` | — | emit `noop` (idempotent) |
| CONNECTED | ANALYZING | varje cykel | ev_status fortfarande connected | beräkna acceptansformel |
| CONNECTED | IDLE | `ev_status == disconnected` | — | emit `noop` |
| ANALYZING | CHARGING | alla R9+R10+R11+R12 uppfyllda | bat_soc >= threshold | emit `ev_start(amps)` |
| ANALYZING | CONNECTED | ej alla villkor uppfyllda | — | emit `ev_decision_reason`, vänta |
| ANALYZING | IDLE | `ev_status == disconnected` | — | emit `noop` |
| CHARGING | CHARGING | varje cykel, villkor fortfarande uppfyllda | — | emit `ev_adjust(amps)` eller `noop` |
| CHARGING | COMPLETING | villkor ej uppfyllda ELLER target nådd | — | emit `ev_stop` |
| CHARGING | ERROR | R1 brott (grid > ±100W) | — | emit `ev_stop` + `incident_alert` |
| CHARGING | ERROR | R2 brott (disconnected under laddning) | — | emit `ev_stop` |
| COMPLETING | IDLE | 1 cykel (15s) grace period klar | — | emit `log_session_done` |
| ERROR | IDLE | alltid | — | emit `ev_stop` (idempotent) |

### 3.3 Villkor: Acceptansformel per cykel (R1–R12)

```python
def _evaluate_acceptance(state: EVDispatchState, cfg: EVDispatchConfig) -> AcceptanceResult:
    """
    Starta/fortsätt EV iff ALLA villkor uppfyllda:
      1. R2: ev_status ∈ PLUGGED_STATUSES
      2. R12: bat_soc >= cfg.bat_ev_ready_threshold
      3. R10: predicted_refill_p10_1_2x >= predicted_bat_deficit
      4. R11: planned_ev_window_min >= cfg.min_ev_session_min (15)
      5. R9:  bat_soc_at_surplus_end >= 100.0
    """
```

### 3.4 Bat-smoothing (R3)

Under CHARGING-tillståndet, varje cykel:

```
bat_discharge_cap = min(cfg.bat_max_smooth_discharge_w, surplus_dip_w)

if pv_power_w < pv_prev_cycle_w - cfg.pv_transient_threshold_w:
    # PV-transient! Täck med bat
    bat_smoothing_w = min(bat_discharge_cap, abs(ev_power_w - surplus_w))
    emit ev_adjust(amps)  # Behåll nuvarande amps
else:
    # Normal: justera EV
    emit ev_adjust(optimal_amps)
```

### 4.3 Fullständig tillstånds-logik (pseudokod)

```python
def evaluate_ev_action(current_state, inp, cfg):
    # ── Grindvakt 0: Feature flag ────────────────────────────
    if not cfg.enabled:
        return EVDispatchResult(EVAction.NOOP, 0, EVDispatchState.IDLE,
                                "ev_dispatch_v2 disabled (feature flag)")

    # ── Grindvakt 1: R1 — Grid ±100W invariant ───────────────
    if abs(inp.grid_w) > cfg.grid_invariant_w and current_state == EVDispatchState.CHARGING:
        return EVDispatchResult(EVAction.INCIDENT_ALERT, 0, EVDispatchState.ERROR,
                                f"R1 brott: grid={inp.grid_w:.0f}W > ±{cfg.grid_invariant_w:.0f}W",
                                is_incident=True)

    # ── Grindvakt 2: R2 — Plug connected ─────────────────────
    if inp.ev_status not in PLUGGED_STATUSES:
        if current_state == EVDispatchState.CHARGING:
            return EVDispatchResult(EVAction.EV_STOP, 0, EVDispatchState.ERROR,
                                    f"R2: plug disconnected under laddning (status={inp.ev_status})")
        return EVDispatchResult(EVAction.NOOP, 0, EVDispatchState.IDLE,
                                f"R2: ej ansluten (status={inp.ev_status})")

    # ── Plug är connected: evaluera acceptance ────────────────
    acc = _evaluate_acceptance(inp, cfg)

    if current_state in (EVDispatchState.IDLE, EVDispatchState.CONNECTED,
                         EVDispatchState.ANALYZING):
        if not acc.accepted:
            return EVDispatchResult(EVAction.NOOP, 0, EVDispatchState.CONNECTED, acc.reason)
        # Alla villkor uppfyllda → start
        amps = _calculate_optimal_amps(inp, cfg)
        return EVDispatchResult(EVAction.EV_START, amps, EVDispatchState.CHARGING,
                                f"Startar EV {amps}A: {acc.reason}")

    if current_state == EVDispatchState.CHARGING:
        if not acc.accepted:
            return EVDispatchResult(EVAction.EV_STOP, 0, EVDispatchState.COMPLETING,
                                    f"Stoppar EV (villkor ej längre uppfyllda): {acc.reason}")
        # Bat-smoothing (R3)
        smoothing_w, amps = _apply_bat_smoothing(inp, cfg)
        if amps == inp.ev_current_amps and smoothing_w == 0.0:
            return EVDispatchResult(EVAction.NOOP, amps, EVDispatchState.CHARGING,
                                    "EV stabil, inga justeringar")
        return EVDispatchResult(EVAction.EV_ADJUST, amps, EVDispatchState.CHARGING,
                                f"EV justerat {amps}A (smoothing={smoothing_w:.0f}W)",
                                bat_smoothing_w=smoothing_w)

    if current_state == EVDispatchState.COMPLETING:
        return EVDispatchResult(EVAction.NOOP, 0, EVDispatchState.IDLE, "Session klar")

    if current_state == EVDispatchState.ERROR:
        # Error → stoppa om aktiv, gå till IDLE
        if inp.ev_enabled:
            return EVDispatchResult(EVAction.EV_STOP, 0, EVDispatchState.IDLE,
                                    "Error state: stoppar EV, återgår till IDLE")
        return EVDispatchResult(EVAction.NOOP, 0, EVDispatchState.IDLE,
                                "Error state: EV redan av, återgår till IDLE")

    # Fallback (bör ej nås)
    return EVDispatchResult(EVAction.NOOP, 0, EVDispatchState.IDLE, "Okänt state → IDLE")
```

## 5. API-spec: core/forecast_integrator.py

### 5.1 Interface

```python
from __future__ import annotations
from dataclasses import dataclass
from datetime import datetime
from typing import Protocol


class ForecastSource(Protocol):
    """Protocol för prognosdatakälla — Solcast, Meteomatics, etc."""

    def get_hourly_forecast_kw(self) -> list[float]:
        """
        Returnerar lista med 24 element (en per timme, från midnatt).
        Enhet: kW (medelvärde per timme).
        """
        ...

    def get_p10_hourly_forecast_kw(self) -> list[float]:
        """
        Returnerar P10-percentil (worst-case) av tim-prognosen.
        P10 = värde som med 90% sannolikhet uppnås eller överträffas.
        """
        ...

    def get_sunset_time(self) -> datetime:
        """Returnerar lokal solnedgångstid för idag."""
        ...


@dataclass(frozen=True)
class ForecastWindow:
    """Resultat av en prognos-integration."""
    total_kwh: float            # Summa energi under fönstret
    p10_kwh: float              # P10-variant (worst-case)
    p10_with_margin_kwh: float  # P10 × cfg.refill_margin_factor (default 1.2)
    from_time: datetime
    to_time: datetime
    hourly_kw: list[float]      # Per-timme detaljer


def integrate_solcast_p10(
    source: ForecastSource,
    from_time: datetime,
    to_time: datetime,
    margin_factor: float = 1.2,
) -> ForecastWindow:
    """
    Integrerar Solcast P10-prognosen över ett tidsfönster.

    Implementationsdetaljer:
    - Hämtar timvis P10-data från ForecastSource
    - Summerar kWh för timmar som överlappar [from_time, to_time]
    - Applicerar margin_factor (R10: 1.2x buffer)
    - Hanterar timezone korrekt (astral per kund-geolokation)

    Args:
        source: Prognoskälla (implementerar ForecastSource Protocol)
        from_time: Startpunkt (timezone-aware)
        to_time: Slutpunkt (timezone-aware)
        margin_factor: Buffert-faktor på P10 (default 1.2 = R10)

    Returns:
```

### 12.1 Bakgrund och user-direktiv

> "Viktigt att EV verkligen laddar efter 22." — User 2026-04-23 20:30
> "Alt B smoothing för natt-bat." — User 2026-04-23 20:58

Historik: Nätterna 2026-04-20, 21, 22 laddades EV INTE trots att bilen var inkopplad.
Root cause: Acceptansformel R9/R10/R11 (surplus/forecast-beroende) är alltid falskt nattetid → EV blockeras.
Fix: Separat entry-path för natt-laddning som kringgår dagtids forecast-regler.

### 12.2 Regel R-natt: Natt-laddning obligatorisk (22:00–06:00)

**Entry-villkor (alla måste vara uppfyllda):**

| Villkor | Check |
|---------|-------|
| R2: Bil ansluten | `ev_status in EV_ACTIVE_STATUSES` |
| SoC-behov | `ev_soc < ev_target_soc` |
| Tidsfönster | `hour_of_day in [22, 06)` (wraps midnight) |
| R12: Bat redo | `bat_soc >= bat_ready_threshold_pct` |

**Exit-villkor (något räcker):**

| Villkor | Reason-sträng |
|---------|---------------|
| Utanför fönster (`hour >= 6`) | `R-natt: outside_window` |
| EV SoC nått target | `R-natt: soc_target_reached` |
| Bil frånkopplad | R2-disconnect (EV_STOP direkt) |

**Skillnad mot dagscykeln:**
- R9 (bat vid solnedgång), R10 (PV-refill), R11 (fönsterlängd) **gäller EJ** nattetid
- Grid är **primär** energikälla — PV-surplus spelar ingen roll
- R1 (grid ±100W invariant) **gäller EJ** nattetid (grid-import för EV är förväntad och stor)
- Natt-laddning sker med `ev_max_amps` (inte surplus-beräknat `_optimal_amps`)

**Observability:**
```
sensor.carma_ev_decision_reason:
  'R-natt: night_window_charge (hour=N)'
  'R-natt: outside_window'
  'R-natt: soc_target_reached'
```

**Config:**
```yaml
ev_dispatch_v2:
  night_mode:
    enabled: true          # Masterswitch (default: true)
    start_hour: 22         # Fönster start (default: 22)
    end_hour: 6            # Fönster slut (default: 6, wraps midnight)
```

### 12.3 Regel R-natt-bat: Bat smoothing-only (Alt B)

**Beslutslogik:** User valde Alt B (smoothing only) — ej Alt A (ingen bat-assist) och ej Alt C (full bat-assist).

**Motivering Alt B:**
- Bat-cycle-cost: ~2–3 öre/kWh
- Opportunity cost (bat-arbitrage förlorad): ~70 öre/kWh
- Natt-grid: ~30 öre/kWh → billigare att köpa från nätet än via bat
- Alt C ("spara 30 öre / kosta 70 öre") = netto −40 öre/kWh → ekonomiskt dåligt
- Bat bevaras på ~100% SoC för morgonpeak 06–09 (dyrast timmar = maximal arbitrage-vinst)
- Smoothing (Alt B) räddar ±100W-invariant vid transienter utan signifikant bat-förbrukning

**Beteende:**

| Scenario | Åtgärd |
|----------|--------|
| Grid-transient ≤ 0W | Bat idle (grid steady) |
| Grid-transient 1–500W | Bat smoother ger exakt transient W |
| Grid-transient > 500W | Bat smoother ger 500W (cap), grid tar resten |
| bat_soc ≤ 80% | Bat smoother avstängs, grid tar allt |
| smoothing.enabled = False | Bat idle oavsett |

**Config-parametrar:**
```yaml
ev_dispatch_v2:
  night_mode:
    bat_smoothing:
      enabled: true       # Masterswitch för bat smoothing (default: true)
      cap_w: 500          # Max bat-urladdning per transient (default: 500W)
      soc_floor_pct: 80.0 # Bat-SoC-golv under natt-EV (default: 80%)
      window_s: 5.0       # Smoothing-tidsfönster (default: 5s)
```

**Precedence:**
- Dagtid: R3 (bat smoothing vid PV-transient, `bat_smoothing_threshold_w`)
- Nattetid: R-natt-bat tar över med striktare cap (500W) och `soc_floor_pct` (80%)

### 12.4 State-machine utökning (ev_dispatch.py)

```
EVDispatchState.night_charging: bool  # True om vi ladddar via R-natt

CONNECTED-fas:
  1. Kör _run_acceptance_formula() (dag-villkor R9/R10/R11/R12)
  2. Om OK → dag-laddning (night_charging=False)
  3. Om misslyckas → kör _check_r_natt() (natt-villkor)
  4. Om OK → natt-laddning (night_charging=True, amps=ev_max_amps)
  5. Om misslyckas → CONNECTED kvar (NOOP)

CHARGING-fas:
  if night_charging:
    - Hoppa över R1 (grid-invariant)
    - R2: plug disconnect → EV_STOP
    - ev_soc >= ev_target_soc → EV_STOP ("R-natt: soc_target_reached")
    - outside window → EV_STOP ("R-natt: outside_window")
    - Annars: NOOP + _compute_night_bat_smoothing()
  else:
    - Befintlig dag-logik (R1 check, R9/R10/R11/R12, bat_smoothing R3)
```
